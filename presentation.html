Interview Note Summarizer

An AI agent that turns raw, hand-typed interview notes into a structured hiring judgment — strengths, concerns, rating, recommendation — built solo, in one day, with Claude Code.

Streamlit Claude API SQLite Docker Kubernetes Live & deployed
The Problem

Raw notes pile up. Decisions don't get easier.

  • Inconsistent input: every interviewer writes notes differently — length, focus, phrasing all vary.
  • Manual re-reading: turning notes into a decision means someone re-reads everything by hand, every time.
  • No consistency: two people reading the same notes can reach different conclusions — nothing forces a structured read.
  • Cost: roughly 10–15 minutes of re-reading and write-up per candidate, per interviewer.
Market Gap — Why This Is Sellable

Nothing else summarizes notes you already typed.

Greenhouse / Lever (ATS)

Free-text scorecard field only. No summarization, no structure, no judgment — just a box to type into.

BrightHire / Metaview / HireVue

Real AI summarization — but only from recorded/transcribed calls, needs integration setup, enterprise pricing.

This tool

Takes notes HR already types by hand and returns a structured, judgment-based summary in seconds — zero setup, zero integration.

That gap — not "it uses AI" — is the actual product wedge: same-day value with no recording pipeline, no call integration, no enterprise onboarding.

Why This Is an Agent, Not a Script

A script matches keywords. This makes a judgment call.

  • Script: keyword-matches — "mentions 'communication'" → tag as soft skill. Brittle, misses anything unstated.
  • Agent: infers concerns the notes only imply, never state outright.
  • Agent: weighs conflicting signals — strong technical answers but a hesitant tone about teamwork — into one overall recommendation.
  • Agent: returns that judgment in the same fixed structure every time, regardless of how messy or differently-worded the input is.

Token & Cost Efficiency

Designed to keep every call small and predictable.

One forced tool-call

Single request per summary — no multi-turn back-and-forth, no agentic retry loop burning extra tokens.

Bounded output

max_tokens=1024 caps every response — cost per summary is predictable, never runaway.

Compact schema

Short string arrays for strengths/concerns, not open-ended prose — the model is guided to stay terse.

Stateless calls

Each summarization is a fresh single-shot request — no growing conversation history re-sent every time.

Right-sized model

Uses Claude Sonnet for a well-scoped extraction task, not an oversized model for simple structured output.

Result

Cost scales linearly with number of summaries — no hidden multiplier from chained calls or context bloat.

Business Case

What this is worth, in plain terms.

  • Time saved: ~10–15 minutes per candidate, per interview, on the re-reading/write-up step alone.
  • Consistency: the same notes get judged the same way regardless of who's reviewing that day.
  • Low infra cost: single SQLite file, containerized app — no database server, no enterprise integration to buy or maintain.
  • Fast to adopt: zero setup, unlike interview-intelligence competitors that require recorded/transcribed calls.
  • Scalable pattern: the same storage + agent design extends directly to shortlist ranking and cross-candidate Q&A — not a rebuild, an extension.
